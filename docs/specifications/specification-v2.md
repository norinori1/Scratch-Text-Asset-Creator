# 改良案

## 1. 「テキストを表示する」関数のwarpオプションをデフォルト有効化

Scratch の「画面を再描画せずに実行（warp）」オプションを、テキスト表示用カスタムブロックの**デフォルトで有効**にする。

### 背景

旧プロジェクト「Text Display」では全カスタムブロックに `"warp": false` が設定されており、
文字を1つ描画するたびに画面が更新されるため、長い文字列を表示する際に体感できるほどの
処理遅延が発生していた。

### 対応方針

生成する `project.json` 内のカスタムブロック定義（`procedures_prototype`）の
`mutation` において、`"warp": "true"` をデフォルト値として設定する。

```json
"mutation": {
  "proccode": "テキストを表示する %s ...",
  "warp": "true"
}
```

ユーザーが意図的に `false` にしたい場合（アニメーション演出等）は、
エクスポート設定UIで切り替えられるオプションを用意する。

## 2. 出力フォーマット選択：ビットマップ（PNG）／ベクター（SVG）

コスチューム画像の出力形式として **SVG（ベクター）** と **PNG（ビットマップ）** を
選択できるようにする。

### 形式比較

| 項目 | SVG（推奨） | PNG |
|------|------------|-----|
| Scratch対応 | ✅ ネイティブ | ✅ |
| ファイルサイズ | 🟢 小さい | 🔴 大きい |
| 拡大時の品質 | 🟢 劣化なし | 🔴 劣化あり |
| 実装難易度 | 🟢 低（パス直接変換） | 普通 |
| 複雑なフォント | △ パスが増大する場合あり | 🟢 安定 |

### 対応方針

- デフォルトは **SVG**。旧プロジェクトも全コスチュームSVG形式であり、継承しやすい。
- `opentype.js` の `glyph.getPath().toSVG()` を使ってパスを直接取得する。
- PNGは `OffscreenCanvas` でラスタライズし、base64→バイナリ変換してZIPに格納する。
- エクスポート設定UIに以下のセレクタを追加する。

```tsx
<select value={renderOptions.outputFormat} onChange={...}>
  <option value="svg">SVG（推奨・軽量）</option>
  <option value="png">PNG（互換性重視）</option>
</select>
```

## 3. 旧プロジェクト「Text Display」の機能移植＋リファクタリング

旧プロジェクトに実装されている機能を新ツールが生成する `.sb3` に移植する。
ただし旧プロジェクトのコードは複雑なため、この機会に設計を見直す。

### 移植する機能一覧

旧プロジェクトの `project.json` を解析した結果、以下の機能を確認した。

| 機能 | 旧実装の状況 | 移植方針 |
|------|------------|---------|
| 文字・数値・記号の表示 | ✅ 実装済み | 継承 |
| X / Y 座標指定 | ✅ 実装済み | 継承 |
| サイズ指定 | ✅ 実装済み | 継承 |
| 色効果（COLOR） | ✅ 実装済み | 継承 |
| 明るさ（BRIGHTNESS） | ✅ 実装済み | 継承 |
| 透明度（GHOST） | ✅ 実装済み | 継承 |
| フォント切替（Serif / Sans） | ✅ 実装済み | 継承（複数フォント対応として拡張） |
| レイヤー順序（layer） | ✅ 実装済み | 継承 |
| 改行 `\n` 検出 | ✅ 実装済み | 継承 |
| 文字間隔設定 | ✅ 実装済み（`Text_Setting` リスト） | 継承・UI化 |
| 軽量化モード | ✅ 実装済み（`Low Detail Mode`） | 継承 |
| 小数点数値の表示 | ✅ 実装済み | 継承 |

### リファクタリング方針

旧プロジェクトの複雑さの根本原因は **`ξ` 区切りの独自エンコード方式** にある。
表示パラメータ（テキスト・座標・色・フォント等）を1つの文字列に詰め込み、
受け取り側でパースする設計になっているため、ブロックが非常に読みにくい。

**旧方式（問題あり）:**
```
// リストの1アイテムに全パラメータを詰め込む
"hello,world! ξ 50 ξ 0 ξ 137 ξ 0 ξ 0 ξ 0 ξ 1 ξ 100"
//  text      x   y  size color bright transp font layer
```

**新方式（シンプル）:**
```
// パラメータごとに素直にリストアイテムを分ける
表示キュー = [
  item 1: text   → "hello,world!"
  item 2: x      → 50
  item 3: y      → 0
  ...
]
```

これにより生成されるScratchブロックが人間にとって読みやすくなり、
ユーザーが手動でデバッグ・改変しやすくなる。

#### 参照先

旧プロジェクトのソースコードは以下に配置されている。

```
docs/specifications/TextDisplay/
├── TextDisplay.sb3
└── TextDisplay.zip（解凍済み）
    └── project.json   ← ブロック定義はここ
```

## 4. テキストアライメント（左揃え / 中央揃え / 右揃え）

旧プロジェクトは左揃え固定だったが、新ツールでは揃え方を選択できるようにする。

### 対応方針

カスタムブロックの引数に `align` パラメータを追加する。

| 値 | 動作 |
|----|------|
| `"left"`（デフォルト） | 指定X座標を左端として右方向に描画 |
| `"center"` | 指定X座標を中心として左右に描画 |
| `"right"` | 指定X座標を右端として左方向に描画 |

`center` / `right` の場合、描画前にテキスト全体の幅を計算してから
クローンの開始X座標をオフセットする前処理ブロックを生成する。

## 5. 自動改行（ワードラップ）

`\n` による手動改行に加え、**表示幅（ピクセル数）を指定すると自動的に折り返す**
オプションを追加する。

### 対応方針

カスタムブロックの引数に `maxWidth` パラメータを追加する。

```
テキストを表示する [text] x: [x] y: [y] 最大幅: [maxWidth]
// maxWidth = 0 のとき折り返しなし（旧互換）
```

折り返し処理の擬似コード：

```
currentWidth = 0
foreach char in text:
  if currentWidth + charWidth > maxWidth:
    改行（column += 1, x = 開始X にリセット）
    currentWidth = 0
  currentWidth += charWidth + letterSpacing
  クローンを生成
```

## 6. テキストクリア用ブロック

「現在表示中の全テキストを削除する」専用のカスタムブロックを生成する。

### 背景

旧プロジェクトでは `delete all clones` ブロックを直接呼ぶか、
表示ループを停止する方法しかなく、ユーザーが独自に実装する必要があった。

### 対応方針

以下のカスタムブロックを `.sb3` に同梱する。

```
define テキストをすべてクリアする
  delete all clones of [FontChar]
```

呼び出し例：

```
when [スペース] key pressed
  テキストをすべてクリアする
```

## 7. 文字描画方式の選択：クローン式 ／ ペン式

`.sb3` 生成時に、文字をScratch上で描画する方式を **クローン式** と **ペン式** から
選択できるオプションを追加する。

### 方式の概要

#### クローン式（旧プロジェクト互換）

各文字コスチュームを持つスプライトのクローンを1文字ずつ生成し、画面上に配置する方式。
旧「Text Display」はこの方式を採用している。

```
FontChar スプライト
  └─ clone × 文字数
       ├─ costume = 対応グリフ
       ├─ x, y    = 表示座標
       └─ size, color, ghost... = 各エフェクト
```

**特徴:**

| 項目 | 評価 |
|------|------|
| Scratch 拡張機能 | 不要（標準ブロックのみ） |
| 文字ごとの個別制御 | ✅ 可能（クローン単体にエフェクトをかけられる） |
| クローン数の上限 | ⚠️ Scratch の上限 300 に注意 |
| 表示後の消去 | `delete this clone` または `delete all clones` |
| 動的な文字更新 | △ 全クローン削除→再生成が必要 |

#### ペン式

Scratch の **Pen 拡張機能**を使い、スタンプまたはペン描画で文字を直接ステージに描く方式。
クローンを生成しないため、長文や頻繁な更新に強い。

```
FontChar スプライト
  ├─ costume を対応グリフに切り替え
  ├─ x, y に移動
  └─ stamp（またはペンで輪郭をトレース）
```

**特徴:**

| 項目 | 評価 |
|------|------|
| Scratch 拡張機能 | ⚠️ Pen 拡張が必要 |
| 文字ごとの個別制御 | ❌ 描画後はステージに焼き付き・消去不可 |
| クローン数の上限 | ✅ 無関係（クローン不使用） |
| 表示後の消去 | `erase all`（ステージ全消去） |
| 動的な文字更新 | ✅ `erase all` → 再スタンプで高速更新 |
| 長文表示速度 | ✅ クローン式より高速 |

### 方式比較まとめ

| 比較項目 | クローン式 | ペン式 |
|---------|-----------|--------|
| 拡張機能 | 不要 | Pen 必須 |
| 文字数制限 | 最大 300 文字 | 制限なし（実用上） |
| 文字単体のエフェクト | ✅ 可 | ❌ 不可 |
| 表示後の部分消去 | ✅ 可（クローン削除） | ❌ 不可（全消去のみ） |
| 高速描画 | △ 文字数が増えると遅い | ✅ 速い |
| 推奨用途 | 少量テキスト・演出重視 | 大量テキスト・スコア表示 |

### 対応方針

エクスポート設定UIに描画方式セレクタを追加する。
選択した方式に応じて、`Sb3Builder` が生成するスクリプトを切り替える。

```tsx
<select value={renderOptions.renderMode} onChange={...}>
  <option value="clone">クローン式（推奨・拡張機能不要）</option>
  <option value="pen">ペン式（高速・長文向け）</option>
</select>
```

ペン式を選択した場合、生成される `project.json` の `extensions` に
`"pen"` を追加する。

```json
"extensions": ["pen"]
```

生成スクリプトの分岐は `ScratchScriptGenerator.ts` 内で行う。

```typescript
if (options.renderMode === "pen") {
  return generatePenScript(glyphs, options);
} else {
  return generateCloneScript(glyphs, options);
}
```

### ペン式スクリプトの擬似コード

```
define テキストをスタンプする (text) x: (x) y: (y) size: (size)
  set size to (size)
  set [curX] to (x)
  repeat (length of (text))
    set costume to (対応グリフ)
    go to x: (curX) y: (y)
    stamp
    change [curX] by (advanceWidth + letterSpacing)
  end

// 消去はステージ全体
define テキストをすべてクリアする
  erase all
```

> **注意:** ペン式の `erase all` はステージ上のペン描画をすべて消去する。
> ペンで描いた他のオブジェクト（背景等）も消えるため、用途を限定して使用すること。
> この制約をUIのツールチップ等でユーザーに明示する。
